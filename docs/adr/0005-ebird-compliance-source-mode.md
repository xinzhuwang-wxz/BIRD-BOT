# eBird 商业合规（P0）+ 数据源模式显式开关

eBird API **本身**（不只 Macaulay 媒体）默认禁商业用途；BirdBot 是付费 SaaS，未获 Cornell 书面许可即上线即违反 ToU（key 持有者对全部使用负责，可被封）。

**决定（P0 红线）**：
- **立即发起** Cornell 商业许可洽谈（`ebird@cornell.edu`）——本地稀有度叙事是核心差异化，值得争取。
- 许可落定前，eBird 能力**只进内部原型/非商业演示，绝不进付费路径**。
- **并行建降级底座**：iNaturalist 公开记录 + taxonomy（免 key）+ 缓存 + 自建/公开物候数据；主链路不绑死非商业数据，许可拿不到也能降级运行。
- Bird Context Service 做「数据源 × 用途 × 授权」矩阵 + 商业用途拦截 + 展示强制署名（`Source: eBird.org`）。

**数据源模式显式开关**（应对可调试性，由用户要求加入）：上下文来源可配置为 `auto | ebird-only | non-ebird-only` 三种**显式模式**，每种可独立调试；当前生效模式与任何降级/熔断/额度耗尽必须被**埋点 surface，绝不静默降级**（修复审计发现的「静默降级」风险）。

**否决**：无许可直接把 eBird 用于商业产品；以及「自动降级但不可见」（不好 debug）。
