# 可观测性 / 埋点为一等公民（day-one）

多租户共享服务需可证明数据去向（GDPR/审计），识别重排/Story 质量是护城河需评测闭环；审计还发现内核 hook 默认 `reraise=False` 会**静默吞错**、Bird Context 降级可能静默。

**决定（由用户要求升级为硬性要求）**：从第一天建结构化埋点，不事后补：
- 每次 LLM / 工具 / 外部 API 调用记录：`tenant/user/device`、逻辑模型名 → 真实供应商、回退链、能力降级、**数据源模式**、token/cost/延迟、数据流向。
- **任何降级 / 熔断 / 额度耗尽 / 数据源切换必须 surface，绝不静默**。
- 为采集/计费 hook 自建告警（不依赖 `logger.warning`，因内核 hook 默认吞错）。
- 为识别重排（命中率/误报）、最佳帧、Story 质量建基准评测集做回归。

**理由**：成本归因、合规审计、调试、质量护城河都依赖它，且事后补齐成本高。
