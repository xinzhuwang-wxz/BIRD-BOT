# 欧盟用户数据的模型/识别 API 目的地策略

目标市场含欧洲。把 EU/UK 用户的鸟图/位置/行为数据发往**无 GDPR 充分性认定的第三国**端点，合规门槛远高于美国 DPF 路径。

**决定**：把「**允许目的地区域 / 合规标签**」做成模型能力注册表的**一等字段 + 硬约束**（不是软权重）。
- **默认**：EU/UK 用户数据只走 EU 区域端点，或已签 DPA+SCC/DPF 的美国端点。
- **默认禁止**流向无充分性认定第三国（含 DeepSeek/Qwen/智谱等中国境内模型），除非专门境内部署 + SCC。
- 与所有 LLM/识别 provider 签 DPA、开 zero-retention/不用于训练、披露 sub-processor；位置**先降精度再跨境**。
- DPF（依赖 FISA 702）视为可变假设，预备 SCC 兜底；具体合规结论**交法务确认**。

**否决**：A（任意端点含中国模型，合规隐患）；C（全本地化/区域化，成本最高、覆盖受限）。
