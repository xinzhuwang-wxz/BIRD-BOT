# 升级识别后端：专用视觉分类 API（与 LLM 层分离）

识别适配层在「不确定 / 冲突 / 会员任务」时升级。**关键澄清**：物种**识别后端**是专用视觉分类器（vision API / 分类模型），与推理/叙事用的 **LLM**（多 provider 路由层，见 [ADR-0007](0007-eu-data-routing.md) 与 LLM 网关）是**两个不同的层**——idea §7 已立场：通用多模态 LLM 不做识别流水线，因为它对物种 ID 不校准、会幻觉、按图成本高。

**决定**：
- 升级识别后端**先用第三方可商用 vision API**（Nyckel / Roboflow / MS Custom Vision 等），先用 API 走；**先用真实欧美喂鸟器场景基准评测再定最终选型**。
- **SpeciesNet**（代码+权重均 Apache-2.0、明确可商用——比 idea 设想更宽松）作设计参考/自训基线（ensemble + geofence + taxonomic rollup 范式）。
- **排除 iNaturalist CV 模型**（私有、禁商业 AI 用途）。
- 升级触发**绑定校准后置信度阈值**（先温度缩放校准再设阈值）；仅会员/不确定/冲突任务在**深度阶段异步**调用，控成本。
- 推理/行为/Story 层用国内外 **LLM 多 provider**（经 LLM 网关路由，受 ADR-0007 区域约束）；多模态 LLM 只接收精选帧 + 结构化证据，**不做物种分类**。

**否决**：用 iNat CV 模型；用通用多模态 LLM 当物种分类器（作设计底座）。
